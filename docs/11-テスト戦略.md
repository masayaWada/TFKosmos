# 11. テスト戦略

### 11.1 単体テスト

- Rust標準のテストフレームワークを使用
- Scanner の各メソッド
- TemplateManager のテンプレート操作
- TerraformGenerator のコード生成
- NamingNormalizer の正規化

テスト実行方法:

```bash
cargo test
```

### 11.2 統合テスト

- スキャン→生成フロー
- テンプレート編集→生成フロー
- API エンドポイント（Axumテストユーティリティ使用）

### 11.3 モック

- AWS/Azure API はモック化（aws-sdk-mock等）
- 実際のAPI呼び出しは統合テストのみ
- tokio-test を使用した非同期テスト
