## ドキュメントの読み方

### 目的別ガイド

| 目的 | 参照ドキュメント |
|------|-----------------|
| プロジェクトを理解したい | 01_基礎 → 02_設計仕様 |
| API仕様を確認したい | 02_設計仕様/API設計 → 04_品質管理/エラーハンドリング |
| UIを変更したい | 02_設計仕様/UI設計 → コンポーネント詳細仕様 |
| 新機能を追加したい | 01_基礎 → 02_設計仕様 → 04_品質管理/テスト戦略 |
| バグを修正したい | 02_設計仕様/ディレクトリ構造 → 05_運用サポート/トラブルシューティング |
| セキュリティを確認したい | 04_品質管理/セキュリティ考慮事項 |
| デプロイしたい | 03_開発ガイド/開発環境セットアップ |
| コントリビュートしたい | 03_開発ガイド/コントリビューションガイド → コミット戦略 |

### ドキュメント間の関係

```text
01_基礎/
    └── システムアーキテクチャ（全体像）

02_設計仕様/
    ├── データモデル（ドメイン層の詳細）
    ├── API設計（API層の詳細）
    ├── UI設計 → コンポーネント詳細仕様
    ├── ディレクトリ構造
    └── 実装詳細

03_開発ガイド/
    ├── 開発環境セットアップ（環境構築・デプロイ・環境変数）
    ├── コントリビューションガイド
    └── コミット戦略

04_品質管理/
    ├── エラーハンドリング
    ├── セキュリティ考慮事項
    ├── テスト戦略
    ├── バックエンドテスト仕様
    └── フロントエンドテスト仕様

05_運用サポート/
    ├── トラブルシューティング
    └── FAQ

06_ロードマップ/
    └── 今後の拡張
```
