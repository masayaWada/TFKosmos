## ドキュメントの読み方

### 目的別ガイド

| 目的 | 参照ドキュメント |
|------|-----------------|
| プロジェクトを理解したい | 01 → 02 → 06 |
| API仕様を確認したい | 03 → 08 |
| UIを変更したい | 04 → 05 |
| 新機能を追加したい | 01 → 05 → 07 → 11 |
| バグを修正したい | 06 → 15 → 11 |
| セキュリティを確認したい | 09 |
| デプロイしたい | 10 → 14 |
| コントリビュートしたい | 16 → 13 |

### ドキュメント間の関係

```
01-システムアーキテクチャ
    ├── 02-データモデル（ドメイン層の詳細）
    ├── 03-API設計（API層の詳細）
    ├── 04-UI設計 ─── 05-コンポーネント詳細仕様
    └── 06-ディレクトリ構造

07-実装詳細
    ├── 08-エラーハンドリング
    └── 09-セキュリティ考慮事項

10-開発・デプロイ
    ├── 11-テスト戦略
    ├── 14-環境設定
    └── 15-トラブルシューティング

12-今後の拡張（ロードマップ）
13-コミット戦略 ─── 16-コントリビューションガイド
```
