# 用語集

TFKosmosプロジェクトで使用される用語の定義です。

---

## TFKosmos固有の用語

### TFKosmos

**TF**（Terraform）と **Kosmos**（ギリシャ語で「秩序・調和」）を組み合わせた名称。混沌とした手動インフラから秩序あるInfrastructure as Codeへの変革を表す。

### スキャン（Scan）

AWS/Azureのクラウド環境から既存のIAMリソース情報を取得する操作。スキャン結果はメモリに一時保存され、Terraformコード生成の入力となる。

### 生成（Generate）

スキャンで取得したリソース情報を元に、Terraformコード（`.tf`ファイル）とインポートスクリプトを作成する操作。

### テンプレート（Template）

Terraformコード生成に使用するJinja2形式のテンプレートファイル（`.tf.j2`）。デフォルトテンプレートとユーザーカスタムテンプレートがある。

### デフォルトテンプレート

`backend/templates_default/` に格納されているシステム標準のテンプレート。変更不可。

### ユーザーテンプレート

`backend/templates_user/` に格納されるカスタマイズ可能なテンプレート。初回生成時にデフォルトからコピーされる。

## AWS IAM用語

### IAM（Identity and Access Management）

AWSのアクセス管理サービス。ユーザー、グループ、ロール、ポリシーを管理する。

### IAMユーザー（IAM User）

AWSリソースへのアクセス権を持つ個別のエンティティ。人間のユーザーまたはアプリケーションを表す。

### IAMグループ（IAM Group）

IAMユーザーの集合。グループにポリシーをアタッチすることで、複数ユーザーに同じ権限を付与できる。

### IAMロール（IAM Role）

特定の権限を持つIAMアイデンティティ。ユーザーやサービスが一時的に引き受ける（AssumeRole）ことができる。

### IAMポリシー（IAM Policy）

権限を定義するJSONドキュメント。アクション、リソース、条件を指定してアクセス許可/拒否を定義する。

### 管理ポリシー（Managed Policy）

AWS管理またはカスタマー管理のスタンドアロンポリシー。複数のエンティティにアタッチ可能。

### インラインポリシー（Inline Policy）

特定のエンティティに直接埋め込まれたポリシー。そのエンティティ専用。

### ARN（Amazon Resource Name）

AWSリソースを一意に識別する文字列。形式: `arn:aws:service:region:account:resource`

### AssumeRole

IAMロールを引き受ける操作。一時的な認証情報を取得してロールの権限を使用できる。

### プロファイル（Profile）

`~/.aws/credentials` に定義された名前付き認証情報セット。複数のAWSアカウントを切り替えて使用できる。

## Azure IAM用語

### Azure AD（Azure Active Directory）

Azureのアイデンティティおよびアクセス管理サービス。現在は「Microsoft Entra ID」に名称変更。

### サービスプリンシパル（Service Principal）

アプリケーションやサービスがAzureリソースにアクセスするためのセキュリティID。

### マネージドID（Managed Identity）

Azure VMやApp Serviceなどに自動的に割り当てられるID。認証情報の管理が不要。

### RBAC（Role-Based Access Control）

Azureのロールベースアクセス制御。組み込みロールまたはカスタムロールを割り当てる。

### テナント（Tenant）

Azure ADのインスタンス。組織のディレクトリを表す。

### サブスクリプション（Subscription）

Azureリソースの課金単位。1つのテナントに複数のサブスクリプションを持てる。

## Terraform用語

### Terraform

HashiCorpが開発したInfrastructure as Code（IaC）ツール。HCL（HashiCorp Configuration Language）でインフラを定義する。

### リソース（Resource）

Terraformで管理されるインフラストラクチャの構成要素（例: `aws_iam_user`, `azurerm_role_assignment`）。

### プロバイダー（Provider）

Terraformがクラウドサービスと通信するためのプラグイン（例: `aws`, `azurerm`）。

### ステート（State）

Terraformが管理するリソースの現在の状態を記録するファイル（`terraform.tfstate`）。

### インポート（Import）

既存のクラウドリソースをTerraformの管理下に置く操作。`terraform import`コマンドを使用。

### HCL（HashiCorp Configuration Language）

Terraformの設定ファイルで使用される言語。`.tf`拡張子。

### モジュール（Module）

再利用可能なTerraformコードのパッケージ。

## 開発用語

### Axum

Rust製の非同期Webフレームワーク。TFKosmosのバックエンドで使用。

### Tauri

Rust + Web技術でデスクトップアプリを作成するフレームワーク。TFKosmosのデスクトップ版で使用。

### minijinja

Rust製のJinja2互換テンプレートエンジン。Terraformコード生成で使用。

### Monaco Editor

VS Codeで使用されているコードエディタコンポーネント。TFKosmosのフロントエンドで使用。

### CORS（Cross-Origin Resource Sharing）

異なるオリジン間でのリソース共有を制御する仕組み。フロントエンドからバックエンドへのAPIアクセスに関連。

### レイヤードアーキテクチャ

アプリケーションを層（レイヤー）に分割する設計パターン。TFKosmosではAPI層、サービス層、ドメイン層、インフラ層で構成。

## サブエージェント管理用語

### サブエージェント

Claude Codeで使用する専用エージェント。特定のタスク（コミット、テスト実行、ドキュメント生成など）に特化した動作を行います。

### エージェント定義

サブエージェントの仕様を記述したファイル群。`agents/subagents/<agent-name>/` 配下に以下のファイルが含まれます：

- `agent.md` - エージェントの役割・ルール・ワークフロー
- `tools.json` - 許可ツールと権限設定
- `model.json` - 推奨AIモデル設定
- `tests.md` - テストケース
- `README.md` - 使い方・ローカル生成手順

### ツール権限

サブエージェントが使用できるツールと権限レベル（read-only / execution）の設定。`tools.json` で管理されます。

### Conventional Commits

コミットメッセージの規約。`<type>(<scope>): <subject>` の形式で記述します。

例: `feat(backend): AWS S3スキャン機能を追加`

詳細は `.claude/rules/commit-strategy.md` を参照してください。
